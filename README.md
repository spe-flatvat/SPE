# SPE (SUZUME PolicyCraft Engine)

**政策を「提言」するのではなく、政策を「設計・検証・破壊・改善」するためのオープンな政策クラフト環境**

---

## SPEとは

SPEは、国家システムの限界を探るための実験的フレームワークです。

正解を提示するものではありません。

極端なシナリオを含むストレステストを通じて、「持続可能な制度設計とは何か」を探求することを目的としています。

- 特定の経済学派やイデオロギーを前提としません
- 制度変更が人・企業・国家の行動に与える影響を、整合的なシナリオで検証します
- 発動条件・出口戦略・自動修正条項を備えた政策設計を行います
- 失敗を前提とした多段階 fallback 設計（N3 → R4 → F3）を特徴とします

---

# オリジンストーリー

最初は、

> **「消費税5%にしたらどうなるんだろう」**

という、ごく素朴な思考実験でした。

しかしシミュレーションを繰り返すうちに、税率だけを変更しても期待した結果にならず、

> **「制度そのものに、まだモデル化されていない何かがある」**

という違和感が生まれました。

その後、シミュレーションモデルを見直した結果、

**控除不能消費税相当額（損税）**

が経済循環へ与える影響を十分にモデル化できていないことが判明しました。

損税解消効果を組み込んだところ、GDP・賃金・税収弾性の挙動が設計思想と整合するようになり、

**「損税は制度に固定化された経済資源（埋蔵金）ではないか」**

という考えに至りました。

SPEでは、この発見を単なる税制論ではなく、

**「制度改革によって市場へ再循環させる経済資源」**

として位置付けています。

---

## シリーズ構成

この思考実験から、複数の政策アーキテクチャが派生しました。

- **F系列**：包摂性と成長を重視した長期成長戦略
- **N3**：規律優先の生存戦略
- **R4**：N3の限界を受けたバランス型リブート
- **F3**：最終縮小・市場原理徹底路線（国家縮退フェーズ）

これらは優劣ではなく、

**異なる国家運営シナリオ**

として設計されています。

---

## 設計思想

SPEが一貫して重視している考え方があります。

> **包摂性と成長戦略のない国家運営は、最終的に国家の持続可能性を失わせる。**

そのため、

- 発動条件
- KPI
- 自動修正条項
- 出口戦略

まで含めて政策を設計しています。

政策は「成功する前提」ではなく、

**失敗した場合まで設計して初めて完成する**

という立場を採っています。

---

## このプロジェクトの特徴

- 発動条件・出口戦略・自動修正条項を明示
- 簡易シミュレーション条件（初期値・リスクイベント・弾性値）を公開
- ストレステストを前提とした政策設計
- AIとの共同設計・共同検証
- 「政策提言」ではなく「政策クラフト」のサンドボックス
